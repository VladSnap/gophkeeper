# Gophkeeper Client Application

Клиентское CLI приложение для безопасного хранения секретных данных и метаданных.

## Структура проекта

```
cmd/gkclient/                 # Точка входа в клиентское приложение
├── main.go                   # Основной файл приложения

internal/client/              # Клиентская логика
├── config/                   # Конфигурация
│   └── config.go
├── models/                   # Модели данных
│   └── models.go             # Secret и Metadata модели
├── storage/                  # Работа с базой данных
│   ├── database.go           # SQLite база данных и миграции
│   ├── migrations.go         # Embedded миграции
│   └── migrations/           # SQL миграции (embedded)
│       ├── 001_create_secrets_table.up.sql
│       ├── 001_create_secrets_table.down.sql
│       ├── 002_create_metadata_table.up.sql
│       └── 002_create_metadata_table.down.sql
├── repository/               # Репозитории для работы с данными
│   ├── interfaces.go         # Интерфейсы репозиториев
│   ├── secret.go            # Репозиторий секретов
│   └── metadata.go          # Репозиторий метаданных
└── service/                  # Бизнес-логика
    └── client.go            # Сервис клиента
```

## Особенности

### База данных
- Использует SQLite для локального хранения данных
- Автоматическое создание базы данных при первом запуске
- Система миграций через **golang-migrate/migrate/v4**
- Миграции встроены в бинарный файл через **embed.FS**
- Поддержка внешних ключей (Foreign Keys)
- Работа с данными через **sqlx** для расширенной функциональности

### Модели данных

#### Secret
- `SecretID` - уникальный идентификатор секрета
- `UserID` - идентификатор пользователя
- `Encrypted` - зашифрованные данные
- `CreatedDate` - дата создания
- `LastUpdatedDate` - дата последнего обновления

#### Metadata
- `MetadataID` - уникальный идентификатор метаданных
- `SecretID` - ссылка на секрет
- `Key` - ключ метаданных
- `ValueHash` - хеш значения
- `ValueEncrypted` - зашифрованное значение
- `CreatedDate` - дата создания
- `LastUpdatedDate` - дата последнего обновления

### Репозитории

Каждый репозиторий предоставляет следующие методы:
- `Create` - создание новой записи
- `GetByID` - получение записи по ID
- `Update` - обновление записи
- `Delete` - удаление записи
- `GetChangedSince` - получение всех измененных записей с указанной даты

Дополнительные методы:
- **SecretRepository**: `GetByUserID` - получение всех секретов пользователя
- **MetadataRepository**: `GetBySecretID`, `DeleteBySecretID` - работа с метаданными секрета

### Сервисный слой

`ClientService` предоставляет высокоуровневые операции:
- `CreateSecret` - создание секрета с метаданными
- `GetSecret` - получение секрета с метаданными
- `UpdateSecret` - обновление секрета
- `DeleteSecret` - удаление секрета и связанных метаданных
- `GetUserSecrets` - получение всех секретов пользователя
- `GetChangedDataSince` - получение всех измененных данных
- `AddMetadata`, `UpdateMetadata`, `DeleteMetadata` - управление метаданными

## Сборка и запуск

### Предварительные требования
- Go 1.24+ 
- Поддержка CGO (для SQLite) или использование pure Go драйвера

### Сборка
\`\`\`bash
go build -o gkclient.exe ./cmd/gkclient
\`\`\`

### Запуск
\`\`\`bash
./gkclient.exe
\`\`\`

### Конфигурация

Приложение может быть сконфигурировано через переменные окружения:

- `GOPHKEEPER_SERVER_URL` - URL сервера (по умолчанию: http://localhost:8080)
- `GOPHKEEPER_LOG_LEVEL` - уровень логирования (по умолчанию: info)
- `GOPHKEEPER_DB_PATH` - путь к базе данных (по умолчанию: ~/.gophkeeper/gophkeeper.db)

## База данных

### Расположение
База данных создается в директории пользователя:
- Windows: `C:\Users\<username>\.gophkeeper\gophkeeper.db`
- Linux/macOS: `~/.gophkeeper/gophkeeper.db`

### Миграции
Приложение автоматически применяет миграции при запуске. Миграции создают:

1. Таблицу `secrets` для хранения зашифрованных секретов
2. Таблицу `metadata` для хранения метаданных, связанных с секретами
3. Таблицу `migrations` для отслеживания примененных миграций
4. Необходимые индексы для оптимизации запросов

## Логирование

Приложение использует структурированное логирование с помощью zap:
- Логи выводятся в консоль и записываются в файл `shortener.log`
- Поддерживаются различные уровни логирования
- Включает информацию о создании/обновлении/удалении данных

## Архитектура

Приложение построено по принципам Clean Architecture:

1. **Слой данных** (`models`, `storage`) - модели и работа с базой данных
2. **Слой репозиториев** (`repository`) - абстракция доступа к данным
3. **Слой сервисов** (`service`) - бизнес-логика приложения
4. **Слой конфигурации** (`config`) - управление настройками
5. **Точка входа** (`cmd/gkclient`) - инициализация и запуск приложения
